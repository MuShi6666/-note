# 后端权限系统

基于 **Sa-Token** 框架实现 RBAC（Role-Based Access Control）权限控制。

---

## 一、整体架构

```
请求 → Sa-Token 全局过滤器 → URL 路径鉴权 → 放行/拒绝
                                   │
                                   ↓
                    StpUtil.checkPermission(url)
                                   │
                                   ↓
                  UserPermissionRedisCacheServiceImpl
                   （实现 StpInterface 接口）
                                   │
                                   ↓
                    查询 Redis 缓存 → 查数据库
```

## 二、核心配置

### Sa-Token 全局过滤器

**文件**: [SaTokenConfigure.java](file:///d:/zky2/easy-agent-platform/easy-platform/easy-platform-base-security/src/main/java/com/hnbits/easyjava/security/config/SaTokenConfigure.java)

```java
@Configuration
public class SaTokenConfigure implements WebMvcConfigurer {

    // 放行路径
    private final static List<String> patterns = Arrays.asList(
        "/", "/favicon.ico", "/auth/form", "/code/**", "/error",
        "/websocket/**", "/docs", "/api-docs", "/swagger-ui/**",
        "/api-docs/**", "/**.html", "/**.css", "/**.js"
    );

    @Bean
    public SaServletFilter getSaServletFilter() {
        return new EasySaServletFilter()
            .addInclude("/**")                                                         // 拦截所有请求
            .addExclude(excludeList.toArray(...))                                      // 放行白名单
            .setAuth(obj -> {
                StpUtil.checkPermission(url);                                          // URL路径鉴权
            })
            .setError(e -> {
                // 异常处理：未登录/无角色/无权限/被封禁
                if (e instanceof NotLoginException) { ... return 401; }
                if (e instanceof NotRoleException) { ... return 403; }
                if (e instanceof NotPermissionException) { ... return 403; }
                if (e instanceof DisableServiceException) { ... return 403; }
            });
    }
}
```

**鉴权策略**: 后端使用 **URL 路径**作为权限标识，每次请求都会检查当前用户是否有该 URL 的访问权限。

## 三、登录认证

**文件**: [AuthAction.java](file:///d:/zky2/easy-agent-platform/easy-platform/easy-platform-base-security/src/main/java/com/hnbits/easyjava/security/action/AuthAction.java)

```java
@RestController
@RequestMapping("/auth")
public class AuthAction {

    @PostMapping("/form")
    public CommonResult<Object> passwordLogin(
            @RequestBody AuthFormRequest request,
            @RequestParam("device") String device) {

        // 1. 根据用户名查询用户
        AuthUserVo authUser = userDetailsService.loadUserFromDb(username);

        // 2. 检查密码
        if (userDetailsService.isRightPassword(password, authUser.getPassword())) {
            // 3. Sa-Token 登录
            StpUtil.login(authUser.getId(), device);

            // 4. 缓存用户信息和权限
            userCacheService.addCache(authUser);                     // 用户信息缓存
            userPermissionCacheService.addCache(authUser.getId());   // 权限缓存

            // 5. 返回 token
            return CommonResult.success(data);
        }
    }
}
```

## 四、权限数据模型

```
es_sys_user（系统用户表）
  │
  ├── es_sys_user_role（用户-角色关联表）
  │     └── es_sys_role（角色表）
  │           ├── es_sys_role_menu（角色-菜单关联表）→ es_sys_menu（菜单表）
```

## 五、权限缓存服务

**文件**: [UserPermissionRedisCacheServiceImpl.java](file:///d:/zky2/easy-agent-platform/easy-platform/easy-platform-base-security/src/main/java/com/hnbits/easyjava/security/service/impl/UserPermissionRedisCacheServiceImpl.java)

此类同时实现了 `IUserPermissionCacheService` 和 `StpInterface` 接口。

### 权限获取流程

```java
// 1. 登录时缓存权限
public void addCache(String userId) {
    List<String> permissionList = this.queryPermissionList(userId);   // 从 es_sys_resource 查
    List<String> roleList = this.queryRoleCodeList(userId);           // 从 es_sys_user_role 查
    List<SysMenuVo> menuList = this.querySysMenuList(userId);         // 从 es_sys_menu 查

    // 存入 Redis（JetCache）
    JetCacheUtil.putCache(KEY_PERMISSION + userId, permissionList, expiration);
    JetCacheUtil.putCache(KEY_ROLE + userId, roleList, expiration);
    JetCacheUtil.putCache(KEY_MENU + userId, menuList, expiration);
}

// 2. Sa-Token 鉴权时调用（StpInterface 实现）
public List<String> getPermissionList(Object loginId, String loginType) {
    // 优先读 Redis 缓存，缓存不存在则查数据库
    List<String> cacheList = this.getPermissionList(loginId.toString());
    if (cacheList != null) return cacheList;
    return this.queryPermissionList(loginId.toString());
}

// 3. 从数据库查询权限
private List<String> queryPermissionList(String userId) {
    List<SysResourceVo> resourceList = sysResourceRepository.queryUserRoleResourceList(userId);
    // 返回所有 resource_url 列表
    // 额外加入 /user/** 和 /auth/** 的放行
    permissionList.add("/user/**");
    permissionList.add("/auth/**");
    return permissionList;
}
```

### Redis 缓存 Key

| 缓存 Key | 说明 |
|----------|------|
| `user_permission_resource_{userId}` | 用户资源权限列表 |
| `user_permission_role_{userId}` | 用户角色列表 |
| `user_permission_menucode_{userId}` | 用户菜单编码列表 |
| `user_permission_sysmenu_{userId}` | 用户菜单树 |

### Sa-Token StpInterface

```java
@Service
public class UserPermissionRedisCacheServiceImpl
    implements IUserPermissionCacheService, StpInterface {

    // StpInterface 接口方法 - 获取权限列表
    @Override
    public List<String> getPermissionList(Object loginId, String loginType) { ... }

    // StpInterface 接口方法 - 获取角色列表
    @Override
    public List<String> getRoleList(Object loginId, String loginType) { ... }
}
```

`StpInterface` 是 Sa-Token 的权限认证接口，在调用 `StpUtil.checkPermission(url)` 时会自动调用此接口获取当前用户的权限列表进行校验。

## 六、路由服务

**文件**: [UserRouteServiceImpl.java](file:///d:/zky2/easy-agent-platform/easy-platform/easy-platform-base-security/src/main/java/com/hnbits/easyjava/security/service/impl/UserRouteServiceImpl.java)

```java
@Service
public class UserRouteServiceImpl implements IUserRouteService {

    @Override
    public List<JSONObject> buildRoute(String userId) throws Exception {
        // 1. 获取用户菜单列表
        List<SysMenuVo> menuList = userPermissionCacheService.getMenuList(userId);

        // 2. 构建顶级菜单
        List<String> topMenuList = menuList.stream()
            .filter(n -> n.getPid().equals("0"))
            .map(SysMenuVo::getId).toList();

        // 3. 递归构建路由树（menu_type = "1" 表示菜单）
        for (String topMenuId : topMenuList) {
            List<SysMenuVo> subMenuList = menuList.stream()
                .filter(n -> n.getMenuFullId().startsWith(topMenuId + "/")
                          && n.getMenuType().equals("1"))
                .toList();
            List<JSONObject> route = this.buildRoute(subMenuList);
            routeList.addAll(route);
        }
        return routeList;
    }
}
```

## 七、URL 鉴权流程示例

```
用户请求: GET /admin/sysuser/get/table
  │
  ├─ Sa-Token 全局过滤器拦截
  │
  ├─ 检查放行路径 → 不在白名单中
  │
  ├─ StpUtil.checkPermission("/admin/sysuser/get/table")
  │     │
  │     └─ StpInterface.getPermissionList()
  │           │
  │           ├─ 从 Redis 获取 [ "/admin/sysuser/**", "/admin/org/**", ... ]
  │           │
  │           └─ 匹配 "/admin/sysuser/**" → 放行 ✅
  │
  └─ 请求到达 Controller
```

**通配符匹配**：`/admin/sysuser/**` 可以匹配 `/admin/sysuser/get/table`、`/admin/sysuser/insert` 等所有子路径。

## 八、前端对应关系

| 后端 | 前端 | 说明 |
|------|------|------|
| `es_sys_resource.resource_url` | `v-access="['xxx']"` 指令值 | 前后端使用相同的URL路径作为权限标识 |
| `es_sys_menu` | 动态路由和菜单 | 后端返回菜单树，前端生成路由 |
| `StpUtil.checkPermission(url)` | `hasAccessByCodes(codes)` | 都是基于资源码的集合判断 |
| Sa-Token 全局过滤器 | 路由守卫 | 都是拦截请求/路由进行权限校验 |
