# 前端权限系统

基于 Vben Admin 5 权限框架 + 后端 Sa-Token URL 拦截。

---

## 一、整体架构

```
登录 → 获取 token → 获取用户信息/权限 → 生成动态路由 → 页面访问权限控制
                                                          ↓
                                                  v-auth指令控制按钮级权限
```

### 核心文件

| 文件 | 作用 |
|------|------|
| `apps/nexai-ui/src/store/auth.ts` | 登录流程、获取用户信息、权限码 |
| `packages/stores/src/modules/access.ts` | accessStore：token、权限码、菜单、路由状态 |
| `apps/nexai-ui/src/router/guard.ts` | 路由守卫：token检查、动态路由生成 |
| `apps/nexai-ui/src/router/access.ts` | 从后端获取菜单树，生成可访问路由 |
| `packages/effects/access/src/directive.ts` | `v-access` 指令注册 |
| `packages/effects/access/src/use-access.ts` | 权限判断逻辑 |
| `packages/stores/src/modules/user.ts` | userStore：用户信息、角色 |

## 二、登录流程

**文件**: [apps/nexai-ui/src/store/auth.ts](file:///d:/zky/web/webrepo/apps/nexai-ui/src/store/auth.ts)

```
1. loginApi(params) → POST /auth/form → 获取 accessToken
2. getUserProfileApi() → GET /user/profile → 获取用户信息和权限码
3. 构建 userInfo { userId, username, roles, ... }
4. accessStore.setAccessToken(token)       ← 存储 token
5. userStore.setUserInfo(userInfo)         ← 存储用户信息
6. accessStore.setAccessCodes(permissions) ← 存储权限码列表
7. 跳转首页
```

**API 响应数据结构** (`/user/profile`)：

```json
{
  "userid": "xxx",
  "usercode": "test188",
  "deptid": "0318",
  "username": "test1",
  "roles": ["ROLE_CSJS"],
  "permissions": ["/admin/sysuser/**", "/admin/org/dept/**", ...],
  "avatar": "..."
}
```

## 三、路由守卫

**文件**: [apps/nexai-ui/src/router/guard.ts](file:///d:/zky/web/webrepo/apps/nexai-ui/src/router/guard.ts)

### setupAccessGuard 流程

```
beforeEach:
  │
  ├─ 核心路由(login/register等) → 直接放行
  │
  ├─ 无 token → 跳转登录页(带 redirect 参数)
  │
  ├─ 有 token 且 isAccessChecked = true → 放行
  │
  └─ 有 token 但未生成动态路由:
      1. fetchUserInfo() 获取用户信息
      2. generateAccess() 从后端获取菜单，生成路由
      3. accessStore.setAccessMenus(menus)
      4. accessStore.setAccessRoutes(routes)
      5. accessStore.setIsAccessChecked(true)
      6. 跳转目标页
```

### 动态路由生成

**文件**: [apps/nexai-ui/src/router/access.ts](file:///d:/zky/web/webrepo/apps/nexai-ui/src/router/access.ts)

```typescript
async function generateAccess(options) {
  return await generateAccessible(preferences.app.accessMode, {
    ...options,
    fetchMenuListAsync: async () => {
      return await getAllMenusApi();  // → GET /sys/menus/route
    },
    forbiddenComponent,               // 403 页面
    layoutMap,                        // 布局组件
    pageMap,                          // 页面组件（自动扫描 views 目录）
  });
}
```

## 四、按钮级权限指令

### v-access 指令

**注册**: [packages/effects/access/src/directive.ts](file:///d:/zky/web/webrepo/packages/effects/access/src/directive.ts)

**两种模式**（由 `preferences.app.accessMode` 控制）：

| 模式 | 用法 | 判断依据 |
|------|------|---------|
| **backend**（默认） | `v-access="['workflow/xg']"` | 根据 `accessStore.accessCodes` 中的权限码 |
| **frontend** | `v-access:role="['ROLE_ADMIN']"` | 根据 `userStore.userRoles` 中的角色 |

**实际项目使用的是 backend 模式**（URL路径匹配），因为 Sa-Token 后端鉴权也是用 URL 路径。

### 权限判断逻辑

**文件**: [packages/effects/access/src/use-access.ts](file:///d:/zky/web/webrepo/packages/effects/access/src/use-access.ts)

```typescript
function hasAccessByCodes(codes: string[]) {
  const userCodesSet = new Set(accessStore.accessCodes);
  return codes.some(code => userCodesSet.has(code));
}
```

### 使用示例

```vue
<el-button v-access="['workflow/xg']">编辑</el-button>
<el-button v-access="['workflow/sc']">删除</el-button>
<el-button v-access="['workflow/sj']">设计</el-button>
```

权限码格式为 `模块/操作`，与后端 `es_sys_resource` 表中的 `resource_url` 字段对应。

## 五、权限码数据结构

```
用户登录
  │
  ├─ 获取角色列表：es_sys_user_role（用户角色关联表）
  │    └─ 角色编码 role_code → "ROLE_CSJS"
  │
  ├─ 获取资源权限码：路由守卫中 fetchMenuListAsync
  │    └─ es_sys_resource（资源表）
  │         └─ resource_url（权限码）→ "/admin/sysuser/**"
  │
  └─ 获取菜单路由：路由守卫中 fetchMenuListAsync
       └─ es_sys_menu（菜单表）→ 构建路由树
```
